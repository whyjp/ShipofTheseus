# 다이어그램 컨벤션

## 한 줄 요약
**인터뷰는 마인드맵 → 유즈케이스 다이어그램 → 시퀀스 다이어그램 순으로 진화한다.** 의도가 두꺼워질수록 더 형식 있는 그림으로 옮겨, 사용자가 같은 그림을 보며 누락·엣지·오해를 잡아낸다. 모든 다이어그램은 마크다운 안의 코드 펜스(Mermaid 또는 PlantUML 텍스트) 로 산출물에 박힌다 — 외부 이미지 의존 금지.

## 진화 단계

| 단계 | 페이즈 | 다이어그램 | 목적 |
| --- | ----- | --------- | ---- |
| ① 발산 | 01 의도 인터뷰 | **마인드맵** | 도메인 명사·동사·예외를 폭넓게 펼침 |
| ② 수렴 | 04 사용자 질의 | **유즈케이스** | 액터·시스템·시나리오 경계 합의 |
| ③ 구체화 | 06 계획 | **시퀀스** | 모듈 간 호출, 페이즈/스프린트 흐름 |

## ① 마인드맵 (페이즈 01)

ⓐ Mermaid `mindmap` 블록.
ⓑ 루트 = 프로젝트 의도 핵심 명사 한 단어.
ⓒ 1차 가지 = "무엇을", "왜", "비목표", "제약", "스테이크홀더", "엣지/예외".
ⓓ 2차 가지 = 1차 가지 아래의 구체 항목.
ⓔ 미인지 예외 후보를 별도 가지(`예외/엣지`) 로 의도적으로 부풀려 사용자에게 "이것도 고려해야 하나?" 질문 트리거.

예시:

````markdown
```mermaid
mindmap
  root((원장-결제))
    무엇을
      잔액 조회
      거래 적재
    왜
      감사 가능 단일 원장
    비목표
      실시간 가격 결정
    제약
      p99 < 200ms
      PII 미저장
    스테이크홀더
      회계팀
      운영자
    예외/엣지
      이중 결제 충돌
      네트워크 partial failure
      백필 시 시간 역전
      통화 단위 변경
```
````

## ② 유즈케이스 다이어그램 (페이즈 04)

ⓐ Mermaid `graph` 또는 PlantUML `@startuml ... @enduml` 둘 중 하나로 통일 (프로젝트 시작 시 한 형식 고정).
ⓑ 액터(사람·외부 시스템) 와 시스템 경계(박스), 유즈케이스(타원/노드).
ⓒ 마인드맵의 "엣지/예외" 가지가 유즈케이스의 *대안 흐름* 으로 매핑되는지 사용자와 확인.

예시 (Mermaid):

````markdown
```mermaid
graph LR
  user((회계팀)) --> uc1[잔액 조회]
  user --> uc2[거래 등록]
  ext((결제 PG)) --> uc3[웹훅 거래 적재]
  uc2 -.대안.-> uc4[이중 결제 거부]
  uc3 -.대안.-> uc5[partial failure 재시도]
```
````

## ③ 시퀀스 다이어그램 (페이즈 06)

ⓐ Mermaid `sequenceDiagram` 블록.
ⓑ 모듈 단위로 그리되 *내부* (서비스 ↔ 도메인 ↔ 어댑터) 와 *외부* (FE ↔ BE ↔ DB ↔ 외부 API) 를 분리해 두 그림으로.
ⓒ 페이즈별 시퀀스도 가능 — 하네스 자체의 페이즈 흐름을 시각화.
ⓓ 각 화살표에 호출 함수명·요청/응답 페이로드 키를 표기.

예시 (모듈 내부):

````markdown
```mermaid
sequenceDiagram
  participant FE as fe/login
  participant API as be4fe/auth
  participant Domain as ledger-core
  participant PG as ledger-adapter-pg

  FE->>API: POST /login {email, pw}
  API->>Domain: Authenticate(email, pw)
  Domain->>PG: FindUser(email)
  PG-->>Domain: User
  Domain-->>API: AuthResult{token}
  API-->>FE: 200 {token}
```
````

## 사용자 검증 의례

각 다이어그램 산출 직후, 사용자에게 같은 그림을 보여주고 두괄식 한 줄 + 객관식 5개 이하로 묻는다 ([`interview.md`](interview.md) 컨벤션 준수):

```
질의: 마인드맵에 누락된 가지가 있습니까?

엣지/예외 가지에 다음 후보를 추가했습니다. 각 항목이 실제로 다뤄야 할 케이스인지 확인 부탁드립니다.

선택지:
1. 모두 다뤄야 함 — 그대로 진행
2. 일부 제거 — 어느 항목인지 알려주시면 갱신
3. 추가 필요 — 어떤 시나리오인지 자유 응답
4. 잘 모르겠음 — 비평가 페이즈로 이월
```

## 안티 패턴

ⓐ 다이어그램 없이 텍스트만으로 의도/계획을 사용자에게 확인 — 누락이 보이지 않는다.
ⓑ 외부 이미지 링크 의존 — diff/오프라인/웹뷰 호환 모두 깨짐.
ⓒ 여러 모듈을 한 시퀀스에 욱여넣음 — 가독성 0. 분리.
