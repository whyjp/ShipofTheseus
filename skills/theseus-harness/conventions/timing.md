# 시간 기록 컨벤션

## 한 줄 요약
**최초 프롬프트 수신 시각을 한 번 기록하고, 모든 페이즈·스프린트 산출물 헤더에 시작·종료·누적 경과·현재 시각을 적는다.** 사용자가 "지금 얼마나 진행됐고, 이 스프린트가 얼마나 걸렸는지" 를 산출물만 봐도 즉시 체감할 수 있어야 한다.

## 시작 시각 기록

페이즈 00 가장 첫 동작에서 다음 파일을 생성한다.

`.ShipofTheseus/<프로젝트명>/timing/start.json`

```json
{
  "started_at_iso": "2026-05-01T17:44:12+09:00",
  "started_at_unix": 1746086652,
  "user_request_excerpt": "..."
}
```

이후 어떤 페이즈도 이 파일을 수정하지 않는다 — 절대 시점은 단일.

## 페이즈 산출물 헤더

모든 페이즈 산출물(`intent/01-intent.md`, `plan/06-plan.md` 등)은 다음 헤더로 시작한다.

```markdown
> **프로젝트:** `<프로젝트명>` · **페이즈:** `01-intent`
> **시작:** `2026-05-01T17:44:12+09:00`
> **종료:** `2026-05-01T17:46:30+09:00`
> **소요:** `2분 18초`
> **누적 경과:** `2분 18초` (최초 프롬프트 이후)
> **현재 시각(작성 시점):** `2026-05-01T17:46:30+09:00`
```

## 스프린트 보고서 헤더

```markdown
> **프로젝트:** `<프로젝트명>` · **스프린트:** `NN`
> **체크포인트 커밋:** `<sha>`
> **시작:** ...  **종료:** ...
> **이 스프린트 소요:** `7분 04초`
> **누적 경과:** `54분 12초`
> **현재 시각:** `2026-05-01T18:38:24+09:00`
```

## 핸드오프 요약

최종 핸드오프 문서에는 다음을 표시한다.

ⓐ 최초 프롬프트 시각
ⓑ 핸드오프 시각
ⓒ 총 소요 (사람이 읽는 단위 — `1시간 23분`)
ⓓ 페이즈별 누적 시간 막대(텍스트 표)
ⓔ 스프린트별 소요 표

## 도구 사용

ⓐ 시간 획득은 `date -u +%FT%TZ` 또는 `date +%FT%T%z` (Bash) 로 통일.
ⓑ 누적 경과는 `(현재 unix - started_at_unix)` 로 계산.
ⓒ 사람이 읽는 포맷은 `1시간 23분 04초` 형태(0 인 단위는 생략 가능).

## 강제 사항

ⓐ 헤더 누락된 페이즈 산출물은 품질 게이트(Phase 09)에서 자동 fail 처리.
ⓑ 사용자에게 보고하는 모든 진행 상황 메시지에 누적 경과를 한 줄 포함 — 침묵으로 시간 보내지 말 것.
ⓒ 스프린트 무한 루프(임계 미달) 동안 매 스프린트 종료 시 사용자에게 누적 경과·현재 시각·다음 시도 의사 여부를 짧게 보고.
