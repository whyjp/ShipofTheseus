# 자율성 컨벤션 (Autonomy First)

## 한 줄 요약
**페이즈 04 의 초기 사용자 질의가 끝나면, 본 하네스는 가능한 모든 결정을 자율로 내리고 산출물에 기록해 사후 리뷰가 가능하게 한다.** 사용자 액션이 *반드시* 필요한 지점은 객관식 답이 비즈니스 함의를 갖는 경우만 — 의도/계획/구현/테스트/회귀 흐름의 일상적 분기는 자율.

## 자율 결정의 사후 리뷰 가능성

모든 자율 결정은 다음 위치에 기록된다 — 사용자가 어느 시점이든 회수해 검토 가능:

ⓐ 결정 자체: 해당 페이즈 산출물의 본문 (예: `intent/05-decisions.md`, `sprints/NN/bisect.md`).
ⓑ 결정의 *이유*: 같은 산출물의 "사유" 섹션 + 점수 근거.
ⓒ 결정의 *대안*: 경쟁이라면 패자 후보 디렉터리(`competitions/.../losers/`) 에 보존.
ⓓ 결정의 *시각*: frontmatter 의 `produced_at` + 시간 메타 헤더.
ⓔ 결정의 *책임 에이전트*: frontmatter 의 `producer_agent`.

따라서 *어떤 자율 결정이라도 핑거프린트 + 시각 + 본문* 으로 추적 가능. 사용자가 사후에 "이건 다른 선택이었어야 했다" 고 지적하면 해당 페이즈만 재실행 (frontmatter 무결성 검증 후 [`contracts.md`](contracts.md) 의 재진입 룰).

## 자율 결정 신뢰도 가드

자율 결정의 *오결정* 위험을 줄이는 메커니즘:

ⓐ **점수 차 임계** — 경쟁 resolve 는 점수 차가 0.02 미만이면 머지, 0.05 이상이면 우승자 채택, 그 사이는 차원별 분석. 임계 사이는 [`competition.md`](competition.md) §4 의 알고리즘으로 결정.
ⓑ **DIP 단독 hard cap** — 후보가 DIP 위반이면 점수 무관하게 탈락. SOLID 의 가장 중요한 원칙은 자율 결정이라도 무시 불가.
ⓒ **회귀 임계** — 자율 결정 이후 점수가 0.05 이상 떨어지면 즉시 페이즈 11 → 결정 자체를 재검토 (자율 위임이라도 회귀 후 재검토는 강제).
ⓓ **확증 회귀** — [`interview.md`](interview.md) §2-2 의 paired_with/consistency 추적 — 사용자 의도와 자율 결정 사이의 정합 검증.
ⓔ **사후 리뷰 알림** — 핸드오프 (페이즈 13) 에 자율 결정 목록과 대안 비교를 포함, 사용자가 "어떤 결정이 자율로 내려졌는지" 한눈에 봄.

## 자율 결정의 점수 디테일 확장

자율 판단의 검증 신뢰도를 높이려면 점수 차원의 디테일이 풍부해야 한다. 본 하네스의 점수는 다음 두 층:

ⓐ **rubric 6 차원** ([`../scoring/rubric.md`](../scoring/rubric.md)) — correctness/scope_fit/solid/coverage/fe_be_parity/e2e_pass.
ⓑ **sub_score 디테일** — 각 차원 내부의 세부 측정. 예:
  ⓑ-1 `correctness` = test_pass_rate × intent_fidelity (게이트 1 결과).
  ⓑ-2 `solid` = 모듈별 SOLID 통과 비율, DIP 위반은 cap 0.5.
  ⓑ-3 `coverage` = min(BE, FE) — avg 가 아닌 min 으로 패리티 강제.
  ⓑ-4 `fe_be_parity` = full/missing_one/missing_two/smoke_only/n/a 5 등급.
ⓒ **메타 자기 평가** ([`../scoring/self_lint.py`](../scoring/self_lint.py) `--score`) — lint 18 + pytest 16 + sample 의 가중 평균. 임계 0.99999.

자율 결정이 신뢰받으려면 *결정 시점의 모든 점수* 가 산출물에 기록 + 핑거프린트로 봉인. 사후 회의에서 "그때 점수가 어땠지" 가 즉시 답 가능.

## 사용자에게 알리는 자율 결정 보고 형식

매 자율 결정 후 사용자에게는 *침묵하지 않는다* — [`timing.md`](timing.md) 의 라이브 보고에 한 줄 추가:

```
스프린트 NN 완료 — 점수 0.997 (직전 0.991). 누적 41분. 자율 결정: 경쟁 resolve (a 채택, Δ +0.06). 다음 스프린트 진행.
```

자율 결정의 사후 회수가 필요하면 사용자가 즉시 정지 명령 (예: "방금 자율 결정 다시 보자") — 정지는 항상 허용. 자율은 *사용자가 이의 없으면* 의 디폴트일 뿐, 항상 회수 가능.

## 사용자 사전 위임 메커니즘

페이즈 04 의 마지막 객관식 질의로 사용자가 자율 위임 수준을 결정:

```
질의: 본 하네스의 자율 판단 위임 수준을 정해주세요.

이번 작업 동안 페이즈 04 외에 사용자에게 추가 질의를 얼마나 자주 보낼지 결정합니다. 회귀 바이섹트와 자동 머지의 자율 처리 가능성에 영향.

선택지:
1. 최대 자율 (회귀·경쟁·머지 모두 자율, 핸드오프에서 일괄 사후 리뷰)
2. 회귀만 ack (회귀 권고는 사용자 ack, 그 외 자율)
3. 모든 분기 ack (보수적 — 회귀, 경쟁 resolve, 자율 업데이트 모두 ack)
4. 다음 스프린트마다 ack (스프린트 종료 시점마다 진행 여부 묻기)
```

답을 `intent/04-autonomy.md` 에 기록. 후속 페이즈는 이 위임 수준을 따른다.

## 안티 패턴

ⓐ 자율 결정을 *침묵 속에* 진행 — 사용자가 무엇이 결정됐는지 모름. **모든 자율 결정은 라이브 보고 1 줄.**
ⓑ 점수 근거 없이 자율 결정 — *왜* 그 후보가 우승했는지 산출물에 명시 안 됨. **점수 + 차원별 분석 필수.**
ⓒ 위임 수준 무시하고 모든 분기를 자율 — 사용자가 "ack 만" 위임했는데 회귀를 자율 처리. **위임 수준은 페이즈 04 답이 source of truth.**
ⓓ 자율 결정 후 산출물 누락 — 사후 리뷰 불가. **frontmatter + 핑거프린트 + 본문 누락 시 게이트 9 fail.**
